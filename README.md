# News Portal - Django Project

## Описание проекта
Новостной портал на Django с реализацией пагинации, фильтрации и CRUD операций. Проект выполнен в рамках учебного задания.

## Выполненные требования

### 1. Пагинация на /news/
- **Ограничение**: не больше 10 новостей на странице
- **Отображение**: видны номера лишь ближайших страниц (±3 от текущей)
- **Навигация**: кнопки перехода к первой (`««`) и последней (`»»`) странице
- **Технология**: Django ListView с paginate_by = 10

### 2. Поиск на /news/search/
- **Фильтры**:
  - По названию (регистронезависимый поиск)
  - По имени автора
  - Позже указанной даты
- **HTML5 календарь**: реализован через `type="date"` в виджете Django Filter
- **Множественная фильтрация**: можно комбинировать все критерии
- **Пагинация**: сохраняет параметры поиска при переходе между страницами

### 3. CRUD операции
#### Для новостей:
- `/news/create/` - создание новости
- `/news/<int:pk>/edit/` - редактирование новости
- `/news/<int:pk>/delete/` - удаление новости

#### Для статей:
- `/articles/create/` - создание статьи
- `/articles/<int:pk>/edit/` - редактирование статьи
- `/articles/<int:pk>/delete/` - удаление статьи

### 4. Особенности реализации
- **Автоматическое определение типа**: поле `post_type` скрыто от пользователя, устанавливается в зависимости от URL
- **Раздельные формы**: новости и статьи создаются через разные представления
- **Валидация**: проверка прав доступа для создания/редактирования/удаления

## Установка и запуск

### Требования
- Python 3.8+
- Django 6.0
- django-filter

### Установка
```bash
# Клонировать репозиторий
git clone <url-репозитория>
cd NewsPortal

# Создать виртуальное окружение
python -m venv venv

# Активировать (Windows)
venv\Scripts\activate
# Активировать (Linux/Mac)
source venv/bin/activate

# Установить зависимости
pip install -r requirements.txt

# Применить миграции
python manage.py migrate

# Создать суперпользователя (опционально)
python manage.py createsuperuser

# Запустить сервер
python manage.py runserver
